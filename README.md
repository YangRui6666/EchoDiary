# EchoDiary 回声日记

> A personal audio memory logger.
>
> 一个将线下声音转化为个人生活日志的记录系统。



EchoDiary 是一个自用的线下生活记录系统。

它通过音频采集设备记录日常交流、临时想法和线下事件，并将录音上传到自建服务器。服务器负责完成语音转文字、内容整理、每日日志生成和历史查询。

## 项目背景

现代生活中，大量信息已经被数字化保存。线上聊天、邮件、文档、代码提交和日程安排都会自然留下记录，之后也容易搜索和回顾。

但线下发生的事情往往不会被很好地记录下来。例如一次面对面的讨论、一段临时产生的想法、一次口头决定，或者一天中零散发生的小事件，很多都会随着时间被遗忘。

EchoDiary 想解决的就是这个问题：把线下生活中容易丢失的信息，转化为可以保存、搜索和回顾的个人日志。

## 设计目标

- 记录线下交流、想法和事件
- 将音频分段保存并上传
- 使用 ASR 将音频转写为文本
- 使用大模型生成每日日志
- 支持按时间、关键词和自然语言查询历史记录

## 系统架构

```text
DJI Mic Mini
    ↓
STM32F405
    ↓
SD 卡分段录音
    ↓
ESP32
    ↓
Wi-Fi 上传
    ↓
自建服务器
    ↓
ASR + LLM
    ↓
每日日志 / 历史查询
````

## 硬件组成

* STM32F405：负责 USB 音频采集和 SD 卡录音
* ESP32：负责 Wi-Fi 上传
* DJI Mic Mini：作为音频输入源
* SD 卡：用于本地缓存音频片段
* 自建服务器：负责转写、总结和查询

## 设备端功能

* USB Audio 音频接收
* RAM 环形缓冲
* SD 卡写入
* 30 秒 WAV 分段
* 待上传文件队列
* 与 ESP32 通信
* 上传成功后标记文件

## 服务器端功能

* 接收音频文件
* 语音转文字
* 按时间线整理文本
* 生成每日 Markdown 日志
* 提取待办和重要事件
* 支持自然语言查询

## 音频格式

Format: WAV
Sample Rate: 16 kHz
Bit Depth: 16-bit
Channels: Mono
Segment Length: 30 seconds


文件示例：

```
/audio/pending/2026-05-12_09-30-00.wav
/audio/pending/2026-05-12_09-30-30.wav

```

## 开发路线

1. STM32F405 SD 卡读写测试
2. STM32F405 枚举 DJI Mic Mini
3. 接收 USB Audio PCM 数据
4. 保存 WAV 文件
5. 实现 30 秒分段录音
6. ESP32 上传音频到服务器
7. 服务器完成 ASR 转写
8. 生成每日 Markdown 日志
9. 实现历史查询

## 设计原则

设备端保持简单可靠：

* 采集
* 缓存
* 分段
* 上传

服务器端负责智能处理：

* 转写
* 总结
* 搜索
* 问答

## 项目定位

EchoDiary 不是一个传统意义上的商业产品，而是一个自用的个人记忆系统。

它的目标是把一天中容易遗忘的线下经历，转化为可保存、可搜索、可回顾的个人日志。
